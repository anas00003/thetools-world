## Data Collection Fields

Copy this table for each keyword cluster.

| Field | Notes |
| --- | --- |
| Competitor name | Brand or website name. |
| Category | Image, PDF, calculator, text, developer, AI, or other. |
| URL | Exact ranking page URL. |
| Tool name | Public tool/page name. |
| Target keyword | Main query the page appears to target. |
| Page title | Full title tag. |
| Meta description | Full meta description. |
| H1 | Exact visible H1. |
| Content sections | Intro, how-to, FAQ, guides, examples, warnings, related tools. |
| FAQ present or not | Yes/no plus question count. |
| Schema present or not | Note SoftwareApplication, FAQPage, HowTo, BreadcrumbList, or none. |
| Ad placement | Header, sidebar, in-content, sticky, footer, none, or unclear. |
| Internal linking pattern | Related tools, category links, breadcrumbs, footer links. |
| Related tools | List exact related tools promoted. |
| Mobile UX notes | Upload area, button clarity, viewport fit, intrusive ads, nav usability. |
| Speed/performance notes | Heavy scripts, lazy loading, image weight, visible delays. |
| Monetization notes | Ads, premium plan, affiliate links, API, subscriptions. |
| Backlink opportunity ideas | Resource lists, education pages, forums, creator workflows, local guides. |
| What TheTools.World can do better | More specific intent, cleaner UX, better privacy wording, fewer distractions, clearer instructions. |

## Review Rules

- Review the top 10 search results for each target keyword.
- Mark whether the query needs a separate tool page or can merge into an existing page.
- Do not create pages for keywords with vague intent, weak uniqueness, or unclear tool feasibility.
- Flag competitors with aggressive ad layouts, misleading buttons, or thin content as patterns to avoid.
